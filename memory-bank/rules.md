# 项目规范

## Always（必须遵守）

1. **每次修改后自动运行测试** — `python -m pytest tests/` 或 `python -m unittest`
2. **提交前确认代码可运行** — `python cli.py --help` 不报错
3. **小步提交** — 每完成一个模块就 git commit
4. **模块化优先** — 扫描引擎不依赖 UI，CLI 和 GUI 各自独立调用引擎
5. **使用 type hints** — 所有函数参数和返回值标注类型
6. **先 Plan 再写代码** — 使用 Plan Mode 讨论方案，确认后执行

## Never（严禁）

1. **不要在扫描引擎中直接 print** — 通过返回值/回调传递结果，由 UI 层决定如何展示
2. **不要在 GUI 线程中执行扫描** — 扫描必须在后台线程，避免界面卡死
3. **不要引入外部依赖** — 只用 Python 标准库
4. **不要硬编码配置** — IP、端口、线程数等都通过参数传入
5. **不要忽略异常** — 所有 socket 操作必须有 try/except，记录错误原因
